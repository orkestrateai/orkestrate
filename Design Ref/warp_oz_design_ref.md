# Warp Oz Design Reference

Detailed visual and UI/UX analysis of the [Warp Oz](https://www.warp.dev/oz) landing page, based on recent design screenshots.

## 1. Visual Theme & Core Aesthetic
- **"Cloud-Native Orchestration"**: The design uses a literal "cloud" metaphor (volumetric clouds/puff-like textures) to represent cloud agents.
- **Micro-Dithering / Noise**: Noticeable dithered gradients transitions from black to deep blue/purple, giving it a technical, retro-modern feel.
- **Glassmorphism**: UI cards for features use semi-transparent backgrounds with subtle borders, allowing the background "clouds" or gradients to peek through.
- **Pixelation/Halftone Patterns**: Areas where the cloud textures meet the dark background often feature a high-contrast dither or pixelated transition.

## 2. Color Palette
- **Primary Background**: Pure Black `#000000`.
- **Accent Purple/Lavender**: Used for the Oz logo and primary branded elements (e.g., `#D1B3FF` style).
- **Secondary Blues**: Soft, airy blues used in the hero "sky" section.
- **High-Contrast White**: Used for primary heading text and call-to-action buttons.
- **Muted Grays**: Used for subheadings and secondary text.

## 3. Typography
- **Serif Headings**: Large, high-contrast serif typeface for main headlines (e.g., "Break out of your shell", "What's in the box"). This gives a premium, sophisticated feel compared to standard monospaced terminal fonts.
- **Sans-Serif Body**: Clean, geometric sans-serif for body text and navigation items to maintain readability and a "technical" look.

## 4. UI Components & Layout
### Hero Section
- **Floating Feature Cards**: Rectangular pills with rounded corners that "float" around the central logo. Each includes an icon and a short value proposition (e.g., "Scale coding agents off your laptop").
- **CTA Buttons**: Rounded-corner buttons. 
    - *Primary*: White background with black text ("Start an agent").
    - *Secondary*: Transparent with white border ("Watch a demo").

### Statistical Section
- **Three-Column Grid**: Large, bold serif numbers (`350K+`, `97%`, `700K+`) followed by smaller sans-serif descriptions.
- **Borders**: Extremely subtle, rounded-corner borders separating the stat blocks.

### Orchestration Map
- **Nodes & Connectors**: A central Oz node connected by thin, gray lines with arrowheads to various services (Web, Integrations, API/SDK, CLI, Warp).
- **Agent Bubbles**: Bubbles on the right representing parallel agents in different states (Searching for bugs, implementing feature, etc.).

### Feature Bento Grid
- **Information Blocks**: The "What's in the box" section uses a horizontal layout of cards.
- **Interactive Previews**: Some cards include snapshots of the Warp terminal interface or mobile integration, often using a "nested container" look.

## 5. Design Philosophies
- **Low-Density, High-Impact**: Plenty of whitespace (or "blackspace") to focus attention on core metrics and visuals.
- **Branded Imagery**: Custom 3D-rendered "cloud" creatures (armadillos/insects) that serve as mascots for the "Oz" service, blending the organic with the digital.
